# HW10-11 – компьютерное зрение в PyTorch: CNN, transfer learning, detection/segmentation

## 1. Кратко: что сделано

- Для части A выбран `Flowers102`, потому что у него есть официальный `train/val/test` split и на малом обучающем наборе хорошо видно, как работают CNN и transfer learning.
- Для части B выбран `OxfordIIITPet`, трек `segmentation`, потому что у датасета естественная бинарная постановка: питомец против фона.
- В части A сравнивались `C1-C4`: базовая CNN, та же CNN с аугментациями, pretrained `ResNet18` head-only и pretrained `ResNet18` partial fine-tuning.
- Во второй части сравнивались два режима постобработки `V1-V2` на pretrained `DeepLabV3_ResNet50`.

## 2. Среда и воспроизводимость

- Python: `3.13`
- torch / torchvision: `2.11.0 / 0.26.0`
- Устройство (CPU/GPU): `mps` для классификации; segmentation-inference выполнялся на `cpu` для консервативной совместимости `torchvision`-модели.
- Seed: `42`
- Как запустить: открыть `HW10-11.ipynb` и выполнить Run All.

## 3. Данные

### 3.1. Часть A: классификация

- Датасет: `Flowers102`
- Разделение: `train/val/test = 1020 / 1020 / 6149`
- Базовые transforms: `Resize(128x128) -> ToTensor() -> Normalize(ImageNet mean/std)`
- Augmentation transforms: `RandomResizedCrop(128) -> RandomHorizontalFlip() -> ColorJitter() -> ToTensor() -> Normalize(...)`

Датасет небольшой по размеру, но содержит `102` класса. На таком объёме особенно хорошо видна разница между обучением модели с нуля и transfer learning. Validation split позволяет корректно выбирать лучшую модель без повторного использования test.

### 3.2. Часть B: structured vision

- Датасет: `OxfordIIITPet`
- Трек: `segmentation`
- Что считается ground truth: бинарная маска питомца, полученная из trimap (`foreground + contour` против фона).
- Какие предсказания использовались: сумма вероятностей классов `cat` и `dog` у pretrained `DeepLabV3_ResNet50`, далее бинаризация и постобработка.
Датасет подходит для бинарной постановки «питомец против фона», а pretrained VOC/COCO segmentation-модель уже знает классы `cat` и `dog`. Поэтому можно не дообучать сеть и сосредоточиться на корректной постановке foreground, визуализации и метриках. Для оценки использован фиксированный поднабор из `60` изображений official test split.

## 4. Часть A: модели и обучение (C1-C4)

- C1 (simple-cnn-base): `SimpleCNN` из четырёх сверточных блоков с `BatchNorm`, без аугментаций.
- C2 (simple-cnn-aug): та же `SimpleCNN`, но train split проходит через `RandomResizedCrop`, `RandomHorizontalFlip` и `ColorJitter`.
- C3 (resnet18-head-only): pretrained `ResNet18`, backbone заморожен, обучается только `fc`.
- C4 (resnet18-finetune): pretrained `ResNet18`, разморожены `layer4 + fc`, что даёт частичное fine-tuning.

- Loss: `CrossEntropyLoss`
- Optimizer(ы): `Adam`
- Batch size: `32` для `C1-C2`, `16` для `C3-C4`
- Epochs (макс): `10` для `C1-C2`, `6` для `C3-C4`
- Критерий выбора лучшей модели: максимум `best_val_accuracy`

## 5. Часть B: постановка задачи и режимы оценки (V1-V2)

### Если выбран detection track

- Не используется.
- V1: `—`
- V2: `—`
- Как считался IoU: `—`
- Как считались precision / recall: `—`

### Если выбран segmentation track

- Модель: pretrained `DeepLabV3_ResNet50`
- Что считается foreground: пиксели питомца на trimap `OxfordIIITPet`; в ground truth объединены внутренняя область объекта и контур.
- V1: базовая постобработка `pet_score >= 0.35`

## 7. Анализ

Простая CNN на `Flowers102` работает заметно хуже pretrained `ResNet18`, что ожидаемо для малого датасета со `102` классами. Даже после `10` эпох `C1` даёт только `0.2529` по `best_val_accuracy`, то есть модели с нуля не хватает данных, чтобы выучить устойчивые визуальные признаки. В `C2` аугментации не помогли. На этом train split комбинация `RandomResizedCrop + ColorJitter` оказалась слишком жёсткой и только усложнила обучение.
Transfer learning здесь оказался решающим. Уже `C3` с замороженным backbone даёт `0.7029`, а partial fine-tuning в `C4` поднимает качество до `0.8892`. Это означает, что ImageNet-признаки хорошо переносятся на `Flowers102`, а разморозка `layer4 + fc` позволяет дополнительно подстроить верхние уровни сети под конкретный датасет.
Разница между head-only и partial fine-tuning хорошо видна по метрикам. В `C3` адаптируется только классификационная голова, поэтому модель ограничена уже готовыми признаками. В `C4` сеть может немного перестроить верхние сверточные представления, и этого хватает для заметного прироста без полного fine-tuning всей `ResNet18`.
Во второй части `mean_iou` подходит для бинарной segmentation-постановки «питомец против фона», потому что эта метрика учитывает и пропуски объекта, и лишние области foreground. `pixel_precision` и `pixel_recall` помогают понять, за счёт чего меняется IoU. В текущем протоколе `V1` лучше `V2` по всем трём метрикам.
Проблема `V2` в том, что более высокий порог и удаление мелких компонент отсекали не только шум, но и полезные части маски. На визуализациях это заметно по ушам, хвостам и узким участкам туловища.

## 8. Итоговый вывод

- В качестве базового конфига классификации я бы выбрал `C4`: pretrained `ResNet18` с partial fine-tuning, потому что он даёт лучший баланс качества и простоты реализации.
- Главное, что показывает эта работа про transfer learning: на малом image dataset он важнее любых локальных доработок самописной CNN.
- По второй части главный вывод такой: даже для простой бинарной segmentation-постановки нужно аккуратно определить foreground, выбрать подходящую метрику и проверять эффект постобработки.
